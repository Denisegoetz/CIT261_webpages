Topic: Designing, Defining, and Triggering CSS3 Transforms without Custom Libraries

CSS3 Transitions Thought Library

Transforming an object can include any of the following: scale(), translate(), rotate(), skew(), and matrix(). The appearance of the element changes before the users eyes, hence the word "transforms". The tranforms can be triggered with JavaScript, a pseudo classes or with an animation.

Transform can be done in 3 different directions, x, y, and z. X transforms manipulate the element along the x-axis or a horizontal line in the center of the element. Y transforms manipulate along the y-axis or a vertical line. Z transforms along the z-axis, or a line going from the screen to the user. The z-axis creates 3d effects which can only be seen if you add a couple of attributes.

Translate Elements

Translate moves things from it's current position to another one. There are several different ways to use translate including translate(5px), translate(5px, 40px), translateX(-30px), translateY(50px), translateZ(20px), and translate3d(5px, 4px, 3px). Negative numbers move the opposite way, on x it moves to the right, and on y it moves up.

Rotating Elements

Rotate turns the object to the specified degrees. Actually there are several options for rotate. The measurements can be written in degrees (deg), radians (rad), percentages (50%) or fraction of a turn (0.3turn). Negative numbers rotate the opposite direction.
