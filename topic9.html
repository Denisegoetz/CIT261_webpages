Topic: Creating CSS3 Transitions and Animations in CSS and triggering them with JavaScript

CSS Transitions

Without transitions, changes to CSS take effect immediately, which can be a jarring user experience. Transitions allow you to make those changes gradually, slowly changing until it's final state. CSS properties can be changed to make a website more understandable or interactive. For example, when a button is hovered over, a change of background color will indicate that it is clickable.

Hover - No Transition
Hover - With Transition

Notice how adding a transition will make the change gradual. Add the transition to the CSS styling rules for the element. You need to specify the property to transition, and the duration. You can transition multiple properties by using a comma separated list. Transition is actually shorthand for transition-property, transition-duration, transition-timing-function, and transition-delay.

transition: width 2s, transform 2s, background-color: 1s 2s, border-radius 3s ease-in;

Additional values that can be added are a transition timing function, and a transition-delay.

  • A transition timing function specifies how the transition will run. For instance, ease-in will start out slow and then speed up, while ease-out will start fast and slow toward the end.
  • Timing delay specifies how many seconds to wait before the transition starts.

Properties you can Transition

You can only transition properties that have an identifiable halfway point, for example

  • background-color, background-position
  • border-color, border-width, border-spacing
  • bottom
  • color
  • font-size, font-weight
  • height left
  • letter-spacing
  • line-height
  • margin
  • max-height, max-width
  • min-height, min-width
  • opacity
  • outline-color, outline-offset, outline-width
  • padding right
  • text-indent, text-shadow
  • top
  • vertical-align
  • visibility
  • width
  • word-spacing
  • z-index

CSS Animations

CSS animations are a step up from transitions because they are more complex but they control a changes in CSS properties as well. They use @keyframes, or timing functions, to control the change over the duration of the change. To create an animation, first write the keyframe, and then call it with the animation property.

To write a @keyframe, first use the keyword @keyframes and give it a name. Then, inside curly braces, create a list of percentages where things will happen in the animation. For instance, 0% is the start of the transition, 50% is halfway through the animation, and 100% is the end of the animation. Then after each percentage, inside curly braces, write the css styling rules that you want to take effect.

Next, in the styling rules for the element, add an animation property. The animation property requires 2 values, the @keyframe name and the duration of the animation in seconds.

This code is what I used for the image slider you see.

/* This one goes forward thru the images then backward */
@keyframes slider {
  0% {left: 0;}
  10% {left: 0;}
  12% {left: -100%;}
  22% {left: -100%;}
  24% {left: -200%;}
  34% {left: -200%;}
  36% {left: -300%;}
  46% {left: -300%;}
  48% {left: -400%;}
  58% {left: -400%;}
  60% {left: -300%;}
  70% {left: -300%;}
  72% {left: -200%;}
  82% {left: -200%;}
  84% {left: -100%;}
  94% {left: -100%;}
  96% {left: 0%;}
}

#slider figure{
   position: relative;
   width: 500%;
   margin: 0;
   left: 0;
  text-align: left;
  font-size: 0;
  animation: 20s slider infinite;
}

Additional values that can be added are animation-timing-function and animation delay (same as on transitions), animation-direction, animation-iteration-count, animation-fill-mode, and animation-play-state.

  • animation-direction can be set to normal, reverse, alternate, etc. and determines what direction the keyframe will run, 0% to 100% or 100% to 0%.
  • animation-iteration-count tells how many times to repeat the animation. from one to infinite.
  • animation-fill-mode will set whether the animation will determine the appearance of the element before and after the animation. The default is none, which will revert back to the original appearance once the animation is done. If you want it to retain the changes, set it to forwards.
  • animation-play-state will determine if the animation is running or paused.

Triggering with JavaScript

CSS Transitions and animations can be applied on pseudo classes (defines a special state of an element) like hover or active. Animations can be set to play all the time or a certain number of times after the page loads, but if you want to have more control over when a transition or animation happens, JavaScript is needed. Notice the text above appear and disapear when you scroll up and down the page.

With JavaScript, transitions and animations can be set to play or pause in response to an event that happens on the page. For example, on the click of a button or scrolling down the page.

scroll

To make a transition or animation play when the user scrolls down the page, use scroll. In this case, the listener will call the scrollAppear() function when a user scrolls on the screen.

Transition triggered by scroll

window.addEventListener('scroll', scrollAppear);
function scrollAppear(){
let hideText = document.querySelector('.hideText');
let textPositiion = hideText.getBoundingClientRect().top;
//get screensize and divide by 2 so the text come up half way (in the middle of the screen)
let screenPosition = window.innerHeight / 2;

if (textPositiion < screenPosition){
  hideText.classList.add('showText');
}else{
  hideText.classList.remove('showText');
  }
}

Animation triggered by Javascript

An animation triggered by a scroll or onclick event can be a little more flashy, but the concept is the same. Add an event listener to JavaScript. Add a @keyframes rule and an animation property to a class in CSS. Create a function that adds and removes classes when the element is clicked.
